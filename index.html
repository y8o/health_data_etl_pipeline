FHIR-lite ETL Pipeline Demo

Overview

This project demonstrates a simple yet powerful ETL (Extract, Transform, Load) pipeline designed to mimic how Electronic Health Records (EHR) data can be standardized and structured for interoperability using a FHIR-lite format.

Dataset

The data originates from the CMS Chronic Conditions Public Use File (PUF). It contains anonymized patient-level data on prevalence of chronic diseases among Medicare beneficiaries.

The original file is in CSV format and includes information such as patient ID, conditions (e.g., Diabetes, Ischemic Heart Disease, Rheumatoid Arthritis), and other health metrics.

What the Script Does

  • Reads in raw clinical data from CSV files.
  • Cleans and normalizes values, including unit conversions and deduplication.
  • Transforms the data into simplified JSON representations that resemble FHIR (Fast Healthcare Interoperability Resources) standards.
  • Outputs structured JSON to the output/ folder, ready for use in an EHR system or further analysis.

Key Technologies

  • Python 3: ETL logic and scripting
  • Pandas: Data manipulation and normalization
  • JSON: Output format for FHIR-lite resources
